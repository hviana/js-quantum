The graphical circuit has the following logic and structure:

Use this description for the AI ​​to reproduce the interface:

IBM Quantum Composer-style circuit editor.

Clean technical interface.
On the left, vertically stacked labels:
q[0]
q[1]
q[2]
c

From each label, a thin horizontal line extends to the right.
The q[...] lines are quantum wires.
The c line is the single classical register line.

The circuit is read from left to right.
Each vertical column represents one logical step in the circuit.
Operations placed in the same column belong to the same step.
Show soft vertical column separators to make the step order visually clear.

Single-qubit gates:
A gate is drawn as a small box or symbol centered on one quantum wire.

Example:
q[0] ───[H]────

Controlled operations:
Use a filled dot on the control qubit.
Use a vertical line connecting control and target.
Use the target symbol on the target qubit.

Example:
q[0] ───●────
        │
q[1] ───X────

If there are multiple controls, stack multiple filled dots vertically on different qubit lines and connect them with one vertical line to the target.

Example:
q[0] ───●────
        │
q[1] ───●────
        │
q[2] ───X────

Measurement:
A measurement symbol is placed on a quantum wire.
A vertical connection goes downward from that quantum wire to the single classical line c.

Example:
q[1] ───[M]────
        │
c    ───╧──────

Classical conditions:
Do not draw separate classical wires like c[0], c[1], c[2].
Only one classical line exists: c.
Conditional execution is shown as an annotation attached to the gate, not as a quantum-style control wire.

Example:
q[2] ───[X | if(c==1)]────

Global vertical columns:
Some elements can appear as full vertical columns affecting all qubit rows at the same horizontal position, such as barriers or phase markers.

Example:
q[0] ───|────(◔)──
q[1] ───|────(◔)──
q[2] ───|────(◔)──

Visual vocabulary:
quantum wire:      ─────
classical line:    ─────
gate box:          [H] [X] [RZ] [M]
control:           ●
target:            X
vertical link:     │
barrier:           |
phase marker:      (◔)
classical cond.:   if(c==k)

Core rules:
- Left side shows q[0], q[1], q[2], c.
- One horizontal wire per qubit, plus one single classical line c.
- Time/order flows left to right.
- Same column = same logical step.
- Vertical links appear only when needed:
  - controlled quantum gates
  - measurement down to c
  - full-column global markers like barriers or phase markers
- Classical conditions are attached to gates as text, not drawn as separate classical control wires.

(qr) => { //qr is SimulationResult

As a code editor, use the Monaco editor via CDNJS.

This is so that each uses the respective sequence:
Execute the classical code (only in the first shot qr is a SimulationResult with an empty map).
Take the result of the classical code and use the return as input for the quantum code with only 1 shot and save the result of that shot.

In the next shot, it will execute:

classical code with the output of the quantum code
quantum code with only 1 shot with the output of the classical code,
etc.

This serves to separate what is quantum from classical, execute both in loops and simulate hybrid systems, which is why the quantum code is executed shot by shot.
